- **タイトル**
GitHub Actions攻撃パターン

- **概要**
Cordycepsと名付けられたCI/CDの脆弱性は、複数のワークフローが連携して権限昇格を許す点にある。攻撃者は無料のGitHubアカウントで悪用可能で、スキャナーは単一ファイルの検査に留まり全体の構成の危険性を検知できない。これにより、正常に見えるパイプラインが実際には侵害されるリスクを孕む。対策は信頼境界の厳格化とソースレベルでのガバナンス強化である。

- **URL**
https://www.bleepingcomputer.com/news/security/the-github-actions-attack-pattern-your-ci-security-scanners-miss/

- **備考**
CI/CDの構成に潜む権限昇格の脅威を分析。攻撃者はpull_request_targetやworkflow_runを悪用し、コード注入やコマンド注入を実行する。これにより、持続的な認証情報の窃取や権限の不正取得が可能となる。推奨対策はワークフローの権限最小化、第三者アクションの固定、手動承認の導入、及びソースレベルでの信頼性検証強化である。将来的にはAI生成ワークフローの増加に伴いリスクが拡大するため、継続的な監視とガバナンスが不可欠である。
