- **タイトル**
GitHub AI支援バグ報告

- **概要**
Wizの研究者がGitHubのgitインフラに高深刻度の脆弱性を発見した。この脆弱性によりリモート攻撃者が単一コマンドでプライベートリポジトリの読み書き権限を取得可能である。GitHubの内部サービスがユーザー入力を盲信し、プッシュオプションの区切り文字を悪用される仕組みである。GitHubは6時間以内に修正を実施し、追加の強化策も導入した。

- **URL**
https://go.theregister.com/feed/www.theregister.com/2026/04/29/github_woah_a_genuinely_helpful/

- **備考**
Wizの発見はGitHubの内部構造解析にAIを活用し、従来数ヶ月かかっていた解析を48時間以内に完了させた点に意義がある。攻撃者はプッシュオプションの区切り文字を悪用し、内部メタデータを改変可能である。GitHubは迅速に対応し、同様の脆弱性の再発防止策を講じた。今後もAI支援による脆弱性発見が増加する可能性が高い。攻撃の被害拡大防止にはログ監視と早期対応が重要である。
