- **タイトル**
OAuth Device Code Phishing

- **概要**
OAuthのデバイスコードフローを悪用したフィッシング攻撃の比較分析。Azureはスコープ制限がなく強力なトークン取得が可能である一方、Googleはスコープを限定し攻撃の影響を抑制している。両者の実装差が攻撃面に大きな違いを生む。

- **URL**
https://www.bleepingcomputer.com/news/security/oauth-device-code-phishing-azure-vs-google-compared/

- **備考**
OAuthのデバイスコードフローを悪用したフィッシング攻撃は、正規の認証機能を利用しつつトークンを不正取得する手法である。Azureは制限が少なく強力なトークンを生成可能で被害が拡大しやすい。対してGoogleはスコープを限定し攻撃の効果を大幅に低減している。これにより同一機能でも実装差が攻撃のリスクを左右することが示された。攻撃者は正規APIを悪用し、ユーザーの認証情報を騙し取りトークンを取得するため、対策としては認証フローの監視、多要素認証の強化、不審な認証要求の検知が重要である。将来的にはOAuth実装のセキュリティ強化が求められる。
